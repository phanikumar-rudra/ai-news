VS Code 1.114 Ships — Copy Final Response and Simplified Codebase Search

Released April 1, VS Code 1.114 focuses on reducing friction in the chat workflow. The standout additions are a "Copy Final Response" command that strips out agent thinking and tool call noise before copying to clipboard, unified codebase indexing that removes the local-vs-remote split, and the ability to reference any previous chat session inside /troubleshoot for after-the-fact diagnosis.

Key Takeaway: Copying a Copilot response to paste into a PR description or doc no longer drags along internal agent logs — Copy Final Response gives you clean Markdown output every time.
  1. Copy Final Response: Right-click any chat response and choose "Copy Final Response" to get only the last Markdown section after all tool calls have run.
  2. Simplified #codebase search: No more local vs remote index management. One state: semantically indexed or not. Copilot builds the index on demand. Workspaces previously using a local non-semantic index will need reindexing.
  3. /troubleshoot with session picker: Use /troubleshoot #session to trigger a session picker and investigate past sessions after the fact, without reproducing the issue.
  4. Video support in image carousel: imageCarousel.chat.enabled now supports video playback in addition to images.
  5. Edit Mode officially deprecated: Removed in 1.125. Temporarily re-enable with chat.editMode.hidden until then.
Real Example: You had a Copilot session last Tuesday where custom instructions were ignored. In today's session, type /troubleshoot #session, pick last Tuesday's session from the picker, and Copilot reads the debug logs and tells you what went wrong — no need to reproduce it.

Fine-Grained Tool Approval — Approve Specific Arguments, Not Whole Tools

A new proposed API in 1.114 lets tool authors scope approval flows to a specific combination of tool plus arguments. Previously, approving the "Run VS Code Command" tool meant approving all commands. Now a tool can prompt "Allow running editor.action.formatDocument?" separately from other commands, so users grant narrow, targeted permissions rather than broad blanket approvals for dangerous operations.

Key Takeaway: Teams building internal Copilot tools can now give users precise control — approve safe file reads without also approving destructive writes or shell execution.
  1. Use the approveCombination field on LanguageModelToolConfirmationMessages to label the specific approval.
  2. When set, VS Code displays a button for the user to approve that exact tool+argument combination.
  3. Approvals are scoped — approving "Allow reading foo.txt" does not approve reading other files.
  4. This is a proposed API in 1.114 — enable it for extension development via the VS Code insiders channel.
Real Example: You build an internal Java dependency-checker Copilot tool. With fine-grained approval, the tool can ask "Allow checking pom.xml for vulnerabilities?" and the developer approves just that file — not all file reads.

Copilot Cloud Agent: Research, Plan, and Branch Without a PR

Copilot cloud agent (formerly "Copilot coding agent") drops the requirement to always create a pull request. It can now generate code on a branch and let you review the full diff before deciding to open a PR. Added alongside this: a plan mode where Copilot generates and presents an implementation plan for your approval before writing any code, and a deep research mode for broad codebase questions grounded in repository context.

Key Takeaway: You can now ask "how does our auth module handle session expiry?" and get a deep research answer grounded in your actual code — or request a plan and review the approach before a single line is written.
  1. Branch without PR: Start an agent session from the Agents tab. Copilot generates code on a branch. Click "Diff" to review changes. Click "Create pull request" only when ready.
  2. Plan mode: Include "create a plan" or "plan first" in your prompt. Copilot generates a structured plan, waits for your approval, then implements.
  3. Deep research: Ask a broad question like "why does our payment service occasionally timeout?" from a Copilot Chat conversation. Copilot kicks off a research session grounded in your repo.
Real Example: Before starting a complex API migration, ask Copilot to "plan the migration of our REST endpoints to GraphQL." Review the 8-step plan, adjust scope, approve — then let Copilot implement while you attend standup.
