VS Code Agents App — Parallel Sessions Across Multiple Repos

VS Code 1.115 ships the VS Code Agents companion app (preview), a standalone application that runs alongside VS Code Insiders and is purpose-built for agent-native development. It lets you kick off agent sessions across multiple repositories in parallel, each isolated in its own worktree. All your existing customizations carry over: custom instructions, prompt files, custom agents, MCP servers, hooks, and plugins. No separate install required — it ships with VS Code Insiders and is accessible from the Command Palette via Chat: Open Agents Application.

Key Takeaway: As a tech lead managing multiple microservices, you can now run a Java API refactoring agent, an Angular UI build agent, and a test-writing agent concurrently across three separate repos — monitoring all of them from one window while you focus on code review.
  1. Install or update to VS Code Insiders.
  2. Open the Command Palette and run Chat: Open Agents Application, or find it in your Start menu / Applications folder.
  3. From the Agents app, start a new session and choose a repository. The app creates an isolated worktree for each session.
  4. Start additional sessions across other repos from the same app window.
  5. Use the context-switching UI to monitor all sessions, review diffs, and create PRs without leaving the app.
Real Example: You need to update a shared API contract across your user-service, payment-service, and Angular frontend simultaneously. Start three parallel agent sessions in the Agents app — one per repo — monitor all three diffs in parallel, and merge when all three are aligned.

Agents Can Now Send Input to Background Terminals

Previously, background terminals in agent sessions were read-only — agents could check output via get_terminal_output but could not respond to prompts. VS Code 1.115 adds the send_to_terminal tool, allowing agents to continue interacting with background terminals. Alongside this, an experimental chat.tools.terminal.backgroundNotifications setting automatically notifies the agent when a background terminal finishes or requires input, eliminating the need for the agent to poll manually.

Key Takeaway: Long-running build or deployment scripts that hit an interactive prompt mid-run no longer stall an agent session — the agent can detect the prompt and respond automatically.
  1. Update VS Code to 1.115.
  2. To enable background notifications: add "chat.tools.terminal.backgroundNotifications": true to settings.json (experimental).
  3. Start an agent session and kick off a long-running terminal command (e.g., a Spring Boot integration test suite).
  4. If the command requires input (password prompt, confirmation), the agent is automatically notified and can respond using send_to_terminal.
Real Example: An agent runs a Docker Compose setup that prompts for a registry password mid-run. With send_to_terminal and background notifications, the agent detects the prompt, sends the credential, and continues the setup — no manual intervention needed.
